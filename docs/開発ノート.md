# 開発ノート（FastAPI + Vue 併存移行）

最終更新: 2025-11-04

## 目的
- 現行の Hono(Cloudflare Workers, SSR) から FastAPI(Backend) + Vue(JS, Vite)(Frontend) 構成へ段階的に移行する
- ローカル開発のDXを向上し、将来の拡張（API化・認証・DB接続）に備える

## 現状
- Hono + Wrangler によるSSR構成が `src/` 以下に存在
- Tailwind などのアセットは `public/` を参照

## 新構成（雛形追加）
- `backend/`: FastAPI（uvで依存管理）
  - `backend/app/main.py` に基本エンドポイント（`/health`, `/api/hello`）
  - CORSは `http://localhost:5173` を許可
- `frontend/`: Vue 3 + Vite（JS）
  - Vite devサーバは `/api` を `http://127.0.0.1:8000` へプロキシ
  - `publicDir` を `../public` に設定し、既存の画像等（`/images/...`）をそのまま参照

## ローカル起動手順
1) Backend
```bash
cd backend
uv sync
uv run uvicorn app.main:app --reload --port 8000
```
2) Frontend
```bash
cd frontend
npm i
npm run dev
```
- ブラウザで `http://127.0.0.1:5173` を開く
- APIの動作確認例: `http://127.0.0.1:8000/api/hello?name=Hiroto` をブラウザで開く、または
  ```bash
  curl "http://127.0.0.1:8000/api/hello?name=Hiroto"
  ```

## デプロイ（Render）
- 方式: Docker（単一サービス）
- Blueprint: `render.yaml`
- Dockerfile: ルートに配置（フロントをビルドして `dist` を同梱）
- ルート配信: FastAPIが `dist` を配信、`/api/*` はAPI、`/health` はヘルスチェック
- CORS: 同一オリジン想定のため不要（ローカル開発用の設定は残置）
